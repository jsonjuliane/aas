A. Accident Detected Flow
System Initialization
Power is supplied to the Raspberry Pi Zero W and all sensors.
Sensors (accelerometer/gyroscope) are calibrated.
GSM and GPS modules initialize.
Continuous Monitoring
The system continuously reads motion data from the accelerometer and gyroscope.
Multi-sensor logic (acceleration + orientation) is applied.
Threshold Detection
If acceleration exceeds the programmed threshold (e.g., ≥ 3g–5g), the system flags a potential accident.
Impact Validation
The system checks:
Sudden spike in acceleration
Abnormal tilt/orientation
If both conditions are met, it confirms a possible accident.
Trigger Alert Sequence
A 5-second countdown is activated.
A pre-recorded voice alert plays through the speaker.
Voice Cancellation Window
The system listens for voice command (e.g., “cancel”) using speech recognition.
If no valid command is detected within 5 seconds, proceed to next step.
Emergency Alert Transmission
The SIM800L sends SMS alerts to registered contacts.
GPS coordinates are included in the message.
System Logging
Event data (time, acceleration, location) is stored for record.
Buzzer
The buzzer sounds when received any text from the Rescue Center of the Barangay.
B. False Detection Flow (No Actual Accident)
System Initialization
Same as normal startup (sensor calibration and module initialization).
Continuous Monitoring
Real-time motion data is continuously analyzed.
Threshold Trigger
Acceleration exceeds threshold due to:
Sudden movement
Helmet drop
Road bump
Preliminary Detection
System flags it as a potential accident.
Countdown Activation
5-second countdown begins.
Audio alert is played.
User Intervention (Cancellation)
Rider responds by:
Saying “cancel” (voice command)
Command Recognition
Speech recognition detects the cancellation command.
Alert Cancellation
Countdown stops immediately.
No SMS is sent.
System resets to monitoring state.
